# Learning Journal: 12 June 2026

## What Happened?
Investigated and fixed a memory leak on the Wikimedia Commons App's upload crop tool. 

## What was Learned?
- React lifecycle hooks or `useEffect` hooks that set up event listeners on DOM elements (like mouse drag tracking for crop handles) MUST return a cleanup function that invokes `removeEventListener`.
- Memory leaks in React apps can be identified by profiling the heap in Chrome DevTools under the Memory panel. I took heap snapshots before and after crop trials to pinpoint the dangling crop handle instances.

## Reviewer Feedback
nicolas-raoul recommended checking if cleanup occurs on unmount. I verified the effect cleanup code, which correctly fixes the leak!
