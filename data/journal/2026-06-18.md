# Learning Journal: 18 June 2026

## What Happened?
Today I finalized the implementation of the Tsishingini (tsw) language metadata and layed out the scripts inside Wikimedia's Universal Language Selector. The patch was successfully merged after two revisions of reviews.

## Why?
To make Wikipedia search tools and editors usable for native Tsishingini speakers, allowing correct fonts and keyboards to load based on language metadata.

## What was Learned?
- Understood Gerrit's patch set workflow better.
- Realized the importance of running the project's internal code generators (e.g. `npm run build:lang` equivalent script) *before* committing new data. If not done, the build breaks on the CI server.
- The ISO 639-3 language subtags must exactly align with SIL databases to prevent rendering errors.

## Reviewer Feedback
ToluAyodele pointed out that the generator scripts weren't executed in my first revision, resulting in stale compiled layouts. I re-ran the scripts, compiled the database, and pushed Patch Set 2 which resolved the CI block.
